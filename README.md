# Aarogya — AI-Powered Voice Medical Appointment Booking Assistant

A fully voice-based AI assistant for hospitals and clinics. Patients can book, manage, and inquire about medical appointments using natural speech — no login required. The experience is designed to feel like a real phone call with a hospital receptionist.

---

## Objective

Build a **public-facing, voice-first** clinic assistant that lets patients:

- Start a voice conversation immediately
- Book doctor appointments
- Check availability
- Reschedule and cancel appointments
- Ask about doctors, specialties, and clinic timings
- Receive responses **only via voice** (TTS audio)

## Features

### Voice-based medical assistant

| Capability | Supported |
|------------|-----------|
| Book appointments | Yes |
| Check doctor availability | Yes |
| Reschedule appointments | Yes |
| Cancel appointments | Yes |
| Doctor / specialty lookup | Yes |
| Clinic hours & contact info | Yes |
| Voice-only responses (TTS) | Yes |
| Continuous conversation (no manual restart) | Yes |

### Voice stack

Built on **[LiveKit Agents](https://docs.livekit.io/agents/)** with production-like voice features:

| Feature | Implementation |
|---------|----------------|
| Voice Activity Detection (VAD) | Silero VAD |
| Turn detection | LiveKit multilingual turn detector |
| Noise cancellation | LiveKit BVC (Background Voice Cancellation) |
| Interruption / barge-in | `min_interruption_duration=0.4` |
| Low-latency streaming | LiveKit WebRTC + streaming STT/TTS |
| Continuous listening | Automatic turn loop until call ends |

### Speech-to-Text (user-selectable)

Patients choose one STT provider in the web UI before starting a call:

| Provider | Notes |
|----------|-------|
| **Deepgram** | Nova-3, recommended for English |
| **Sarvam AI** | Saarika v2.5, strong for Hindi & Telugu |
| **ElevenLabs** | Scribe v1 |

### Text-to-Speech

| Language | TTS provider |
|----------|--------------|
| English | ElevenLabs Flash v2.5 |
| Hindi | ElevenLabs Flash v2.5 |
| Telugu | Sarvam Bulbul v2 (ElevenLabs has no real-time Telugu) |

### Language support

Three languages, selected before the call starts:

- **English**
- **Hindi**
- **Telugu** (regional language)

The assistant conducts the **entire conversation in the selected language** — booking, cancellation, rescheduling, and clinic queries all work in all three languages.

### AI responsibilities

The AI agent:

- Understands natural-language appointment requests
- Extracts intent: doctor/specialty, date & time, action (book / cancel / reschedule / query)
- Asks follow-up questions when information is missing
- Confirms details verbally before finalizing bookings
- Maintains conversation context for the active voice session

**Strict rules (enforced in prompts + backend):**

- Must **not** hallucinate doctor availability — all slots come from Postgres
- Must rely on backend clinic data and scheduling rules before confirming
- If information is missing: *"I don't have that information."*
- If no clinic data exists: *"No clinic data configured. Please contact support."*

### LLM providers

Configurable via `LLM_PROVIDER` in `agent/.env`:

| Mode | Behavior |
|------|----------|
| `auto` (default) | Gemini for quality; falls back to local Ollama on rate limits |
| `gemini` | Cloud Gemini only |
| `mistral` | Mistral AI (`MISTRAL_API_KEY`) |
| `ollama` | Local Ollama only |

## Voice interaction flow

```
User speaks into microphone
        ↓
Voice Activity Detection (VAD)
        ↓
Speech-to-Text (STT) — user-selected provider
        ↓
AI + backend scheduling logic (tool calls against Postgres)
        ↓
System validates doctor availability and booking rules
        ↓
Response generated by LLM
        ↓
Text-to-Speech (TTS)
        ↓
Voice playback in browser
        ↓
Automatically continue listening for next input
        ↓
Repeat until conversation ends
```

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│  Browser (Next.js — Vercel)                                 │
│  • Language + STT picker                                    │
│  • LiveKit WebRTC audio                                     │
│  • POST /api/livekit-token → issues room token              │
└──────────────────────────┬──────────────────────────────────┘
                           │ WebRTC audio
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  LiveKit Cloud                                              │
│  • Real-time audio rooms                                    │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  Python voice agent (LiveKit Build)                         │
│  VAD → STT → LLM + tools → TTS → playback (barge-in)       │
│                                                             │
│  Tools: list_specialties, find_doctors, check_availability, │
│         book_appointment, cancel, reschedule, lookup,       │
│         clinic_information                                  │
└──────────────────────────┬──────────────────────────────────┘
                           │ SQL
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  Neon PostgreSQL                                            │
│  • Clinics, doctors, schedules, appointments                │
│  • Partial unique index prevents double-booking             │
└─────────────────────────────────────────────────────────────┘
```

| Package | Role | Deploy to |
|---------|------|-----------|
| `web/` | Browser UI + LiveKit token API | **Vercel** |
| `agent/` | Python voice worker + scheduling | **LiveKit Cloud** |
| Database | Schema + seed data | **Neon** |

There is **no shared root `.env`**. Each package loads its own config. They connect through **LiveKit Cloud** (WebRTC), not direct HTTP between apps.

## Quick start (local)

### Prerequisites

- Node.js 18+ and pnpm
- Python 3.11+
- API keys: LiveKit, Deepgram, Sarvam, ElevenLabs, Gemini (or Mistral), Neon Postgres

### One-shot setup

```bash
./scripts/setup.sh
```

This installs dependencies, runs tests, initializes the database, and builds the web app.

### Run locally (two terminals)

**Terminal 1 — web**

```bash
cd web
cp .env.example .env.local    # paste LIVEKIT_* from agent/.env
pnpm install && pnpm dev       # http://localhost:3000
```

**Terminal 2 — agent**

```bash
cd agent
cp .env.example .env           # fill in all API keys + DATABASE_URL
./dev.sh dev                   # starts the LiveKit voice worker
```

Open http://localhost:3000, pick a language and STT provider, and start talking.

> **Note:** The agent is a LiveKit **worker**, not a FastAPI/uvicorn server. Do not run `uvicorn main.py`.

### Validate environment

```bash
cd agent && .venv/bin/python scripts/validate_env.py
```

### Run tests

```bash
cd agent && .venv/bin/python -m pytest
```

## Database

Schema and seed data are defined in `agent/clinic/orm/`:

- `tables.py` — SQLAlchemy models (clinics, doctors, schedules, appointments)
- `seed.py` — demo clinic data

Initialize from the agent folder:

```bash
cd agent
.venv/bin/pip install -e .
.venv/bin/python scripts/init_db.py
```

Availability is computed from doctor schedules, blocked periods, existing appointments, clinic timezone, and slot duration. A partial unique index prevents concurrent double-booking.

## Deployment

### Web → Vercel

1. Connect the GitHub repo to Vercel
2. Set **Root Directory** to `web`
3. Add env vars from `web/.env.example`

### Agent → LiveKit Cloud

```bash
# Install LiveKit CLI
brew install livekit-cli
lk cloud auth

# From agent/
cd agent
lk agent create          # links folder, creates livekit.toml
lk agent update-secrets --secrets-file .env --overwrite
lk agent deploy
```

**Required secrets in LiveKit Cloud** (from `agent/.env`):

| Secret | Notes |
|--------|-------|
| `LIVEKIT_URL`, `LIVEKIT_API_KEY`, `LIVEKIT_API_SECRET` | Same as Vercel |
| `DATABASE_URL` | Neon Postgres |
| `DEEPGRAM_API_KEY`, `SARVAM_API_KEY`, `ELEVEN_API_KEY` | STT + TTS |
| `GEMINI_API_KEY` or `MISTRAL_API_KEY` | LLM |
| `LLM_PROVIDER` | Use `gemini` or `mistral` in cloud (Ollama unavailable) |
| `CLINIC_TIMEZONE` | `Asia/Kolkata` |

### Database → Neon

Run `scripts/init_db.py` once against your Neon instance (from your laptop or CI).

Both web and agent **must use the same** `LIVEKIT_URL` / API key / secret.

## Agent tools (scheduling backend)

The voice agent calls these tools against Postgres — never inventing availability:

| Tool | Purpose |
|------|---------|
| `list_specialties` | List available medical specialties |
| `find_doctors` | Find doctors by specialty or name |
| `check_availability` | Open slots for a doctor on a date |
| `book_appointment` | Book after verbal confirmation |
| `lookup_appointment` | Find booking by reference + phone |
| `cancel_appointment` | Cancel an existing booking |
| `reschedule_appointment` | Move to a new date/time |
| `clinic_information` | Hours, phone, address |
